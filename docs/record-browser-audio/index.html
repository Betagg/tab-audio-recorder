Workflow

How to record browser audio

  1. Open the browser audio source

    Play the website, course, webinar, podcast, or web player in a Chrome tab.

  2. Start Dolphin on that tab

    Click the extension while the audio tab is active so Dolphin captures the right source.

  3. Listen while recording

    Keep the tab audible and avoid muting or reloading the source while the recording is running.

  4. Stop and trim

    Stop when the useful part is done and trim silence or extra audio with the waveform editor.

  5. Download the MP3

    Save the final recording locally through Chrome downloads.

Compared with desktop recording

Record the browser tab, not the whole computer

Traditional audio recording tools often need a virtual audio cable, system audio routing, or a separate editor. That can be overkill when the sound is already playing in one Chrome tab.

Dolphin keeps the capture focused on the browser tab and saves the result as a local MP3.

Limits

What browser audio recording cannot guarantee

Dolphin records audio you have the right to capture from supported Chrome tabs. Some protected or DRM-based streams may block browser tab capture.

If the site reloads, changes players, or mutes the tab, stop and start a fresh recording from the active source.

FAQ

Common questions

How can I record browser audio in Chrome?

Open the Chrome tab that plays the audio, start Dolphin from the toolbar, keep the tab audible, stop when finished, trim the recording, and download the MP3.

Does Dolphin record all browser tabs at once?

No. Dolphin focuses on the active Chrome tab after you start recording.

Is this the same as system audio recording?

No. Dolphin records audio from the active browser tab. It does not capture full system audio or microphone input.

Can I record protected browser audio?

Some protected or DRM-based audio may not be available to browser tab capture.

Where is the browser audio file saved?

Dolphin saves the finished MP3 locally through Chrome downloads.
